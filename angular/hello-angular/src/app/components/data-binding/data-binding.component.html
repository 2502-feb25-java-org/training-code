Data Binding

Angular supports data binding, which is a mechanism for coordinating parrts of a template(HTML) with parts of a component(TS)

There are a few different types of data binding

String Interpolation
String interpolation uses double curly braces to display data represented in the component.ts or to evaluate expressions. There are many things we can do with this, however it does have some restrictions, such as we cannot create new objects in template expressions Examples:
  • Math: 2 + 2 = {{ 2 + 2 }}
  • Expressions: 10 > 2 ? = {{ 10 > 2 ? 'yes' : 'no'}}
  • My name is {{ name | uppercase }}
  • Time: {{ time | date:'medium' | uppercase }}
  • Here, we are exploring money pipes, 1000000 dollars => {{ 1000000 | currency:'CAD' }}
  • Here, we are using our custom square root pipe to get the square root of 10000 {{ 4 | sqroot }}
Event Binding
This is another form of data binding which links actions directly to DOM elements -- we do this by adding our events to our elements in the opening tag as a special attribute 
  <element (event) = "function()>
{{count}}
Property Binding
This allows us to apply a property to a DOM element dynamically. There are many types of property binding. 
  <element [property] = "variable">
Two-way data binding
Thus far, our data bindings have bound either our template to our typescript ( HTML --> TS = event binding), or our typescript to our HTML (TS --> HTML = property binding and template expressions). But how can we establish a relationship that goes both ways between template and TS. We want to no longer have to obtain data from input field values - we simply want our input fields to represent some data. We do this with something called a 'Directive', and a special one at that This directive is ngModel. In order to use this, however, our @NgModule must import the FormsModule
